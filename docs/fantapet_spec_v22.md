## 마감 버튼

- 상단 네비게이션 바 우측
- 마감 상태에서는 **마감해제** 버튼으로 전환
- 마감해제 시 사유 입력 + 담당자 선택 필수 (전체 담당자 선택 가능)

### 마감 차단 항목 (미완료 시 마감 불가, 설정에서 on/off)
1. 내일생산불러오기 처리 안 됨
2. 오늘 동결건조 발주 행 확인 처리 안 됨
3. 오늘 입고 예정일인데 완료/취소 처리 안 됨
4. 미확인 자동 재포장 로그 있음
5. 생산 로그 미확인 항목 있음 (`autoRepack:trigger`/`autoRepack:diff` 제외 — 차단 ④ 전담)
6. 사무 로그 미확인 항목 있음 (`minStock:alert` 제외 — 경고 ②③ 전담)
7. 오늘 생산 있었는데 계란 출고 입력 안 됨

### 경고만 띄우는 항목 (마감 가능, 확인 후 진행, 설정에서 on/off)
1. 내일 생산 입력 없음 (휴일 제외한 다음 영업일 기준)
2. 봉투 최소재고 미달 항목 있음
3. 원육 최소재고 미달 항목 있음

### 마감 클릭 흐름 (v21 명시, v22 팝업 표시 정책 추가)
- 차단 항목 1건 이상 → 차단 팝업, 마감 진행 불가
- 차단 0건 + 경고 1건 이상 → 경고 목록 + "그래도 마감하시겠습니까?" 확인 팝업 → Yes면 담당자 선택으로 진행
- 차단/경고 0건 → 곧바로 담당자 선택

### 차단 팝업 표시 정책 (v22 신설)

#### 제목 분기
- 클릭 대상이 **오늘**인 경우: `오늘 마감을 진행할 수 없습니다`
- 클릭 대상이 **과거 영업일**(미마감 상태로 누적된 경우): `지난 영업일 마감이 처리되지 않았습니다`
- 두 경우 모두 본문에 차단 항목 목록 표시

#### 항목 표시 형식
각 차단 항목에 **번호 + 항목명 + 갯수**, 가능하면 **세부 정보** 1줄씩 표시:
```
⑥ 사무 로그 미확인 항목 있음 1건
- bag:adjust — 봉투 수동조정 — -1장 / 담당: 윤은희
```

추상적 표시(`사무 로그 미확인 항목 있음`) 금지. 운영자가 무엇을 처리해야 하는지 즉시 식별 가능해야 함.

#### 권한 문구
- 마지막 안내 문구: `위 항목 처리 후 마감 버튼을 눌러주세요.`
- v20 권한 변경 이후 마감은 전체 계정 가능. **"QC 계정에서"** 같은 특정 계정 명시 금지.

#### 경고 팝업도 동일 패턴 (운영 후 fix 후보)
v22 시점 경고 팝업은 갯수만 표시 (`② 봉투 최소재고 미달 1건 있습니다`). 세부 정보 표시는 운영 후 별도 묶음에서 차단 팝업과 통일.

### 설정 데이터 구조 (v21 신설)
- `settings/closingFlags` 단일 문서로 관리
- 키 (boolean):
  - `blockTomorrowProd`, `blockFrozenOrder`, `blockScheduleDue`, `blockAutoRepack`,
    `blockProdLog`, `blockOfficeLog`, `blockEggOut`
  - `warnNoTomorrowProd`, `warnBagMin`, `warnMeatMin`
- 문서 미존재 시 전부 ON으로 처리
- 설정 UI는 17절 참조

## 1번 화면 (왼쪽 전체)

- 상단에 현재 표시 날짜 + 요일
- 기본값: 오늘
- 캘린더 날짜 클릭 시 해당 날짜 생산으로 변경
- 상단 **내일생산불러오기** 버튼 (잘 보이게 상단 배치)
- 완료 후 대표+사무실 계정에만 **내일생산불러오기 취소** 버튼 표시

**내일생산불러오기 차단 조건 (한 팝업에 전체 목록 표시)**
1. 입고 예정 미완료 (오늘 입고 예정인데 완료/취소 처리 안 됨)
2. 원료 재고 부족 (필요량/현재량 수치 표시)
3. 봉투 재고 부족 (필요량/현재량 수치 표시)
4. 오늘 자동 재포장 확인 미완료
5. 오늘 계란 출고 미입력 (오늘 생산이 있는 경우)

차단 팝업 형식:
```
내일생산불러오기를 진행할 수 없습니다.
아래 항목을 확인해주세요.

① 닭가슴살 원료 재고 부족 (필요 10kg / 현재 3kg)
② 500g 봉투 재고 부족 (필요 200장 / 현재 50장)
③ 오리정육 자동 재포장 확인 미완료
④ 오늘 계란 출고가 입력되지 않았습니다

                              [확인]
```

**차단 항목 없을 때 성공 흐름**
- 담당자 선택 팝업 표시 (주임 그룹 드롭다운)
- 확인 클릭 → 트랜잭션 실행
  - 다음 영업일 생산 원육 + 봉투 재고 차감
  - stockLedger 기록
  - 다음 영업일 생산 카드 연노랑 배경으로 표시
  - 마감 차단 항목 해제
- 하루 1회만 가능 (runDate 기준), 재클릭 시 "내일생산불러오기는 하루 1회만 가능합니다" 팝업
- 오늘 생산 없어도 버튼 누를 수 있음

**내일생산불러오기 취소 버튼 (대표+사무실만)**
- 완료 후에만 표시
- 클릭 시 취소 사유 + 담당자 입력 필수
- 취소 시 재고 전체 롤백 + 버튼 다시 활성화
- 마감 차단 항목 "내일생산불러오기 처리 안 됨" 다시 활성화
- activityLogs에 취소 담당자 + 사유 기록

**새로고침 버튼**
- 사무에서 생산 변경/취소/추가 시 사용
- 클릭 시 기존 차감 롤백 + 변경된 생산 기준 재차감
- 새로고침 시 사무 로그에 기록

**생산 카드**
- 오늘 생산: 흰색 배경
- 다음 영업일 생산 (완료 후): 연노랑 배경
- 카드 내용: 레시피명(대상 포함) + 회차 / 생산단위+단위명 / 생식이면 박스수 / 동결건조면 봉지수·빵판수·동결판수
- 카드 순서: 레시피 순번 디폴트, 드래그앤드롭으로 당일 수동 조정
- 동일 레시피 동일 생산수량은 드래그앤드롭 고정 (순서 변경 불가)
- 생산 없으면 "오늘 생산 없음" / 다음 영업일 생산 없으면 "다음 영업일 생산 없음"
- **크게보기** 버튼 → 전체 화면 모달 (보기 전용, 반응형)

## 3번 화면 (오른쪽) — 로그

위아래 분할.

### 생산 로그 (위)
1. 📅 이벤트 당일
2. 📦 입고 예정일 도래 (확인 필수)
3. 생산 자동 재포장 (확인 필수)

5. 입고 완료 차이 없음
6. 입고 완료 차이 있음 ⚠️ (확인 필수)

8. 마감: "04/27 23:15 마감 / 담당: 홍성희"
9. 마감해제: "04/27 23:20 마감해제 / 담당: 홍성희 / 사유: OO"

### 로그 공통 규칙
- 확인 버튼 있는 로그: 확인 전까지 상단 고정
- 확인 완료 로그: 최근 10건만 표시
- 날짜 표시: 오늘은 시간, 어제는 "어제", 그 전은 날짜(04/20)
- 다음날 로그인 시 전날 마감 로그 사무 로그에 표시
- **전체보기** 버튼 → 모달 (10일치 스크롤)
  - 모달 상단 **히스토리** 버튼 → 전체 기간 로그 + 검색
- 로그 보관: 누적 보관 (조회 기본값 1년, 설정에서 변경 가능)
- 설정에서 로그 항목 순서 드래그앤드롭으로 변경 가능 (생산 로그 / 사무 로그 각각)

### 자동 재포장 확인 팝업
- 시스템 자동 재포장 수량 (수정 불가) — 확인 전까지 이 값으로 잔량 표시
- 시스템 수량의 권위 값은 lot/로그의 `surplusG` (g 단위)
- 모달에는 참고용 환산 개수(`surplusG / processedUnitWeightG`) 같이 표시
- 실제 재포장 수량 입력칸 — **개수로 입력**, g는 `actualCount × processedUnitWeightG`로 자동 환산 표시
- 담당자 선택 (주임 그룹)
- **확인** / **취소**
- **권한**: `office` 차단, `admin/production`만 [확인] 처리 가능 (v20 추가)
- 확인 후 실제 입력값으로 lot `remaining` 보정 (g 단위)
- 차이 있으면 `autoRepack:diff` 활동 로그 발행 (REQUIRES_ACK_KEYS 포함, 확인 필수)
- 실제 수량 0개 입력 시 별도 confirm 팝업 → lot `remaining=0`, `closed=true`

### 캘린더 이벤트 등록/수정/삭제 권한 (v20 추가)
- `admin/office`만 가능, `production` 차단
- 차단 방식: 모달 진입 시점에 `canEditCalendar` 분기로 처리

## 전체 레이아웃

## 왼쪽 — 레시피 목록

- 상단 **레시피 신규 추가** 버튼
- 드래그앤드롭으로 순번 수정 (생식/동결건조 따로)
- 레시피명 앞에 대상 붙여서 표시 (공용은 레시피명만)
- 카테고리 태그 (생식/동결) 표시
- 활성 체크박스 즉시 반영
- 비활성 레시피 흐리게 표시
- 비활성화 시 연결 봉투 자동 해제
- 재활성화 시 봉투 다시 선택 필요
- 레시피 배경색 설정 (통계 차트 색상 연동)
- 삭제 버튼 없음, 비활성/활성 토글만 (v20: `deleteRecipe` 함수 완전 제거됨)

## 오른쪽 — 레시피 상세

상단 우측 **수정** / **저장** 버튼.
- 기본: 읽기 전용
- 레시피 수정은 수정 이후 생산부터만 적용 (기존 생산 기록 스냅샷 유지)

### 기본 정보
- 레시피명 / 비고
- 카테고리 선택 (생식 / 동결건조) — 필수, 변경 불가
- 대상 선택 (고양이 / 강아지 / 공용) — 필수, 기본값 없음
- 연결 봉투 드롭다운 (해당 카테고리 봉투만, 1:1 연결)
- 레시피 배경색 선택 (색상 피커)

### 생식 레시피 추가 설정
- 팩당 중량 입력 (g)
- 박스 계산: 총중량 ÷ 팩당 중량 올림 ÷ 20

### 동결건조 레시피 추가 설정
- 생산 단위 1당 제품 봉지수
- 생산 단위 1당 빵판 수
- 생산 단위 1당 동결판 수
- 분리 작업 필요 여부 체크박스

### 원료 테이블

| 원료명 | 기준 중량(g) | 단위 | 생산단위 | 단위명 | 자동재고차감 | 재고연동 |
|---|---|---|---|---|---|---|

- 셀 클릭 → 인라인 input 활성화
- 생산단위: 라디오 버튼 (1개만 선택)
  - 생산단위 원료에 단위명 자유 입력 가능
  - 생산단위 원료에 기준 중량(g) 입력 (1단위당 중량)
- 재고연동 토글: ON이면 원육 종류 드롭다운, OFF면 직접 타이핑
  - 기본값: ON
  - 토글 변경 시 해당 행 원료명만 초기화
- 자동재고차감: 체크박스
- 원료 순서: 드래그앤드롭
- 물 16,000g 이상 → 물1/물2 균등 분할 (생식/동결건조 공통)

**엑셀 복붙**
- 탭·줄바꿈 파싱해서 행/열 자동 분배
- 원료명이 원육 종류 목록에 있으면 재고연동 자동 ON, 없으면 OFF
- 힌트: "엑셀에서 복붙하면 자동으로 채워집니다"

# 8. 생산 입력 (메뉴 2번)

## 전체 레이아웃

좌우 분할. 왼쪽 3/4 / 오른쪽 1/4.

## 왼쪽 3/4 — 생산 목록

**최상단 얇은 줄**
- 날짜 선택기 (오늘+미래만) + 캘린더 아이콘
- 휴일 지정 체크박스 (메인 캘린더와 동기화)
- 선택한 날짜 + 요일 표시

**생산 카드 목록**
- 가로 나열 (flex wrap), 한 줄에 4~6개
- 레시피 순번 디폴트, 드래그앤드롭으로 당일 수동 조정 가능
- 동일 레시피 동일 생산수량은 드래그앤드롭 고정
- 같은 날 같은 레시피 여러 번 생산 가능
  - 회차 자동 부여 (1회, 2회...)
  - 동일 레시피 동일 수량이면 "1차, 2차"로 표시
  - 카드 삭제 시 회차 자동 재정렬
  - 수동 회차 변경은 드래그앤드롭으로
- 카드 클릭 → 오른쪽 폼에 불러옴
- 카드 내 삭제 버튼 → confirm → 삭제
- 생산 없으면 "오늘 생산 없음" 텍스트
- **크게보기** 버튼 → 전체 화면 모달 (보기 전용, 반응형)

## 오른쪽 1/4 — 생산 입력 폼

1. 생산 담당자 드롭다운 (사무, 디폴트값 설정 가능)
2. 레시피 드롭다운 (활성 레시피만, 순번대로) — 저장 전 변경 가능
3. 생산단위 입력칸 (단위명 표시)
4. 원료 테이블 (자동 계산, 읽기 전용, 스크롤)
5. 생식이면 박스수 / 동결건조면 봉지수·빵판수·동결판수 (참고용)
6. **저장** 버튼 (하단 고정)
7. **생산지시서 복사하기** 버튼 (오른쪽 상단 끝)

**생산지시서 복사하기**
- 생산 입력 화면 오른쪽 상단에만 있음 (메인 대시보드에는 없음)
- 항상 복사 가능
- 클릭 시 팝업
- 자동 생성 양식:
```
※ YY/M/D 요일 생산
- 고양이 생식 : 치킨140, 더블민스80
- 강아지 생식 : 치킨13.5, 램2.5
- 동결건조 : 고양이 주식치킨캣25, 강아지 덕캣20, 치킨텐더20
[비고내용]
```
- 동결건조 안에서 고양이 → 강아지 → 공용 순서 디폴트, 드래그앤드롭으로 변경 가능
- 동일 레시피 동일 수량 2번이면 "치킨140, 치킨140 (1차, 2차)" 표시
- 비고 없으면 비고줄 생략
- **복사하기** 버튼 → 클립보드 복사 + "복사 완료!" 팝업

**저장 차단 조건**
- 미확인 자동 재포장 로그 있으면 차단
- 원료 재고 부족 시 노란색 경고만 표시, 저장 허용 (차감 없음)

**생산 카드 삭제**
- 내일생산불러오기 전 삭제: 그냥 삭제 (차감 없었으니 롤백 없음)

**저장 후 수정**

- 내일생산불러오기 전: 생산단위 수정 저장만 수행 (재고 차감 없음)

**재고 차감 시점**
- 생산 입력 저장 시: 재고 차감 안 됨
- 메인 대시보드 **내일생산불러오기** 버튼 클릭 시: 다음 영업일 생산 원육+봉투 재고 차감

5. 냉동창고도 부족하면 내일생산불러오기 차단

- 계란 관리 화면 상단에서 최소재고(개) 직접 설정
- 최소재고 미달 시 사무 로그 ⚠️ 경고 (매일 반복)
- eggStock 문서에 minimumQty 필드로 저장

**FIFO 잔량 펼침 표시 (v21 신설)**
- 상단 요약 바 "현재 재고 N개" 카드에 토글 추가
- 기본 닫힘, 클릭 시 펼침. 새로고침 시 닫힘
- 펼치면 카드 아래에 FIFO 가정 분해 표시
- FIFO 차감 규칙:
  - eggLogs 전체를 `createdAt` 시간순 재생
  - `type === 'in'` → 별도 lot 생성 (입고)
  - `type === 'out'` → 오래된 lot부터 차감
  - `type === 'adjust'` && `qty < 0` → 오래된 lot부터 차감
  - `type === 'adjust'` && `qty > 0` → "조정" lot으로 별도 표시
  - 같은 날 입고 여러 건도 `createdAt`/id 기준으로 별도 lot 유지
- 표시 형식: 오래된 입고일부터, `날짜 담당자 +수량 (입고/조정) → N개 남음`
- 잔량 0인 lot은 표시 안 함
- 정합성 검증:
  - FIFO 분해 합계와 `eggStock.currentQty` 비교
  - 차이 있으면 경고 문구 표시
  - 입고 lot보다 먼저 기록된 출고/감소가 있으면 배분 불가 수량 표시
- 안내 문구: "* FIFO 가정 표시 — 실제 출고 lot은 추적되지 않음 (참고용)"

# 13. 동결판 재고 (메뉴 7번)

## 전체 레이아웃

테이블 형식. 날짜 세로 / 제품 가로.

**상단 요약**
- 빵판 총재고 / 실리콘 총수량 / 현재 총 동결판

**테이블 구조**
- 날짜 + 담당자 (1명만 선택)
- 제품별: 빵판(환산) / 동결판 — 2개 열씩
- 빵판 재고 / 동결판 재고 (날짜별 표시) / 비고

**동결판 날짜별 추적**
- 동결판 입고 시 날짜 자동 기록
- 재고 표시:
```
치킨텐더: 15판 (26/04/25)
          10판 (26/04/27)
```
- 발주 차감 시 FIFO (오래된 날짜부터)

**행 두 종류**
- 작업 행은 날짜당 1개. 제품 추가 시 옆으로 열이 붙는 구조. 같은 제품이면 수량 수정 또는 합산. 입력된 제품만 표시 (0은 안 보임).
- 발주 행 (노란색): 날짜+"발주" / 제품별 돌릴 판수 수동 입력 / 총합 45판 기준 색상

**발주 행 구조**

| 제품명 | 돌릴 판수 |
|---|---|
| 치킨텐더 | 10 |
| 주식치킨캣 | 35 |
| 총합 | 45 ✅ |

- **발주 확인** 버튼 → 동결판 재고 FIFO 차감 + ledger 기록 + 마감 차단 해제
- 확인 전 발주행: **발주 삭제** 버튼 → confirm 후 삭제 (재고 차감 없었으니 롤백 없음)
- 확인 완료된 발주행: **발주 취소** 버튼 → confirm 후 ledger 기반 동결판 재고 롤백 + 마감 차단 항목 "오늘 동결건조 발주 확인 처리 안 됨" 재활성화 + activityLogs 기록
- 발주 날짜: 실제 동결건조 돌리는 날짜 (과거 날짜 등록 불가)

**환산 공식 (설정에서 변경 가능)**
- 빵판 → 동결판: × 4/9
- 빵판 → 실리콘: × 4

**당일 행만 수정/삭제 가능. 이전 날짜는 수동 재고 조정만.**

**버튼**: + 작업 행 추가 / + 발주 행 추가 / 담당자 관리(선임+사무)

날짜 기준: 입고된 날짜. 같은 날짜 같은 제품이면 한 행에 합산. 입고 날짜가 다른 것만 여러 행으로 표기.

**상단 요약 바**
- 제품별 현재 재고 합계: 분리X / 분리O / 소분X (전체 합계)

**재고 종류**
- 분리X: 분리 작업 전 재고
- 분리O: 분리 완료, 포장 대기
- 소분X: 분리 불필요 제품, 포장 대기
- 소분O 개념 없음 (출고하면 재고에서 사라짐)

**테이블 열**
- 분리 필요 제품: 분리O / 분리X 열 쌍
- 분리 불필요 제품: 소분X 열만

**버튼 4개**

**① + 원물 입고**
- 제품명 (동결건조 레시피 드롭다운)
- 분리 필요 여부에 따라 분리X 또는 소분X로 자동 입고

**② + 분리 작업**
- 제품: 현재 분리X 재고 있는 것만 (잔량 표시)
- 수량: 분리X 재고 초과 시 경고
- 저장 → 분리X 차감 + 분리O 증가 (남은 분리X 유지)

**③ + 출고**
- 제품: 현재 분리O 또는 소분X 재고 있는 것만
- 재고 종류 선택: 분리O / 소분X
- 수량: 재고 초과 시 경고

**④ + 수동 조정**
- 재고 종류 선택 드롭다운: 분리O / 분리X / 소분X
- 조정 유형: + 증가 / - 감소
- 수량 입력
- 사유 + 담당자 필수

**당일 행만 수정/삭제. 이전은 수동 조정만.**

**담당자 관리**: 선임+사무

# 15. 입고 예정관리 (메뉴 9번)

**입고 예정 등록**
- 예정일 / 구분(원육/봉투/계란)
  - 원육 → 원육 종류 드롭다운
  - 봉투 → 봉투 종류 드롭다운
  - 계란 → "계란" 자동 고정
- 발주 수량 + 단위 (원육: g/kg / 봉투: 장/박스 / 계란: 개/판)
- 발주 담당자 (사무)
- 입고메모
- 같은 품목 중복 입고 예정 등록 불가 (수정으로 중량 변경)
- **추가** / **취소**

**목록 테이블**

| 예정일 | 구분 | 품목명 | 발주수량 | 실제수량 | 발주담당자 | 입고담당자 | 상태 | 입고메모 | 완료메모 | |
|---|---|---|---|---|---|---|---|---|---|---|

- 상태: 예정(회색)/완료(초록)/취소(빨강)
- 예정일 오름차순, 같은 날짜는 최신 등록이 위
- 지난 날짜 예정 상태면 노란색 경고
- 완료/취소 항목 기본 숨김 (토글로 표시)
- 수정 버튼 있음 (v21 명시: 예정 상태에서만 노출)
- 입고 예정 미완료 시 메인 대시보드 **내일생산불러오기** 버튼 차단
  - 차단 메시지: "[품목명] 입고 예정이 미완료입니다. 입고 예정관리 처리 후 내일생산불러오기가 가능합니다."

**수정 처리 (v21 신설)**
- 등록 모달 재사용 (`editingDocId` 분기로 동작)
- 수정 가능 필드: 예정일 / 발주수량 / 단위 / 발주 담당자 / 입고메모
- 수정 불가 필드: 구분(type) / 품목(itemId) — 변경 필요 시 취소 후 재등록
- 중복 차단은 자기 자신 제외 후 기존 규칙 적용
- 마감 가드: 원래 예정일 또는 새 예정일 중 하나라도 마감이면 수정 차단
- `activityLogs`에 `schedule:edit` 기록 (담당자 포함, 사유 없음)
- **권한**: `admin/office`만 가능

**완료 처리**
- 발주 수량 수정 불가 표시
- 실제 수량 입력칸 (발주 수량 옆)
- 입고 담당자 (주임, 필수)
- 완료메모
- **완료** → 해당 재고 자동 입고 (완료 처리한 날 기준)
  - 원육 → 냉동창고
  - 봉투 → 봉투 재고
  - 계란 → 계란 관리 (단위: 개)
- 차이 있으면 사무 로그 ⚠️ + 확인 필수
  - 로그: "[품목명] 입고 완료 ⚠️ 발주 N → 실제 N / 담당: OO"

**완료 처리 되돌리기 정책**
- 완료 후 취소/수정 불가
- 실수 발견 시 수동 재고 조정으로 처리

**취소 처리**
- 취소 사유 입력 필수
- confirm 팝업 후 취소 처리

# 17. 설정 (메뉴 12번)

**담당자 관리**
- 그룹 3개 (선임/주임/사무) 담당자 추가/수정/삭제
- 그룹명 수정 가능
- 담당자 삭제 시 기존 이력 담당자명 그대로 유지
- 메뉴별 담당자 그룹 체크박스 설정 (v21: UI 운영 시작 후 추가 예정. 운영 시작 시점에는 spec 5절 매트릭스가 코드 하드코딩으로 적용됨)

**시스템 설정**
- 동결건조 1회 최대 판수 (기본 45판)
- 빵판 → 동결판 환산 공식 (기본 ×4/9)
- 빵판 → 실리콘 환산 공식 (기본 ×4)
- 계란 1개당 노른자 참고 중량 (g)
- 생식 박스당 팩 수 (기본 20팩)
- 물 분할 기준값 (기본 16,000g)
- 로그 조회 기본 기간 (기본 1년)

**로그 순서 설정**
- 생산 로그 항목 순서 드래그앤드롭으로 변경
- 사무 로그 항목 순서 드래그앤드롭으로 변경

**마감 체크 항목 on/off**
- 데이터: `settings/closingFlags` 단일 문서 (4절 참조)
- 문서 미존재 시 전부 ON으로 처리, UI에서 토글 변경 시 해당 키만 `false`로 저장
- 차단 7개
  - 내일생산불러오기 처리 여부 (`blockTomorrowProd`)
  - 오늘 동결건조 발주 확인 처리 여부 (`blockFrozenOrder`)
  - 오늘 입고 예정 완료/취소 처리 여부 (`blockScheduleDue`)
  - 미확인 자동 재포장 로그 (`blockAutoRepack`)
  - 생산 로그 미확인 항목 (`blockProdLog`)
  - 사무 로그 미확인 항목 (`blockOfficeLog`)
  - 오늘 생산 있었는데 계란 출고 미입력 (`blockEggOut`)
- 경고 3개
  - 내일 생산 입력 없음 (`warnNoTomorrowProd`)
  - 봉투 최소재고 미달 (`warnBagMin`)
  - 원육 최소재고 미달 (`warnMeatMin`)
- 경고 ②③ off 시 매일 1회 최소재고 자동 발행도 건너뜀

**마스터 활성/비활성 토글 UI 일관성 (v22, 운영 후 통일)**
- v22 시점에 마스터별로 토글 UI 위치 다름:
  - 원육 (9절): 종류 관리 모달의 활성 컬럼에서 일괄 관리
  - 봉투 (11절): 각 봉투의 수정 모달 안 토글
  - 동결제품 (12절): 각 제품의 수정 모달 안 토글
- 운영 1~2주 안정화 후 별도 묶음에서 한 가지 패턴(권장: 원육 방식)으로 통일

# 18. 기술 설계 원칙

## 운영 신뢰 모델
인증된 내부 사용자만 사용하며, Firestore rules는 역할별 접근 제한과 주요 필드 보호 중심으로 적용한다. 복잡한 업무 정합성은 앱 로직과 트랜잭션으로 보장한다.

## 타임존 규칙
- 모든 날짜: Asia/Seoul 기준 YYYY-MM-DD로 저장
- Firestore 저장 시 timestamp는 UTC, 표시는 Asia/Seoul 변환
- "오늘", "내일", "다음 영업일", "18개월 후", "마감" 전부 이 기준

## 다음 영업일 계산 규칙
- 토/일 자동 빨간색 처리 (자동 공휴일)
- 추가 공휴일: 설정 화면에서 수동 등록/삭제
- 다음 영업일 = 토/일 + 수동 공휴일 제외한 가장 가까운 다음 날짜

## 단위 저장 원칙
- 원육: g 기준으로 DB 저장 (kg 입력 시 자동 변환, 표시는 kg)
- 봉투: 장 기준으로 DB 저장 (표시는 박스 변환)
- 계란: 개 기준으로 DB 저장
- 통계 표시: kg/박스로 변환해서 표시

## 공통 Audit 필드
모든 문서에 공통으로 포함:
- `createdAt`: 생성 시각 (UTC timestamp)
- `updatedAt`: 수정 시각 (UTC timestamp)
- `createdBy`: 생성 계정 uid
- `updatedBy`: 수정 계정 uid

## 음수 재고 허용 범위
- 수동 조정: 마이너스 가능 (원육/봉투/계란 전부), 빨간색 표시
- 자동 차감 (내일생산불러오기/전처리/재포장/동결제품입고): 부족 시 차단
- 생산 입력 저장: 재고 부족 시 경고만 표시 (노란색), 저장 허용

## 재고 문서 구조 원칙
- master 문서 (bagTypes, meatStocks 등): 현재 재고 수량 집계 필드 포함
- log 문서 (bagLogs, activityLogs 등): 입고/차감/조정 이력만 저장 (불변)
- ledger는 master 문서의 집계 필드를 차감 대상으로 참조

## 재고 차감 Ledger (롤백 전략)
- 방식: 차감 스냅샷 저장
- **재고를 자동으로 증가/차감하는 모든 트랜잭션은 stockLedger에 기록**
- 대상: 내일생산불러오기, 생산 새로고침, 전처리, 재포장, 입고예정완료, 동결제품입고, 발주확인
- 수동 조정은 ledger 대신 activityLogs로만 기록 (취소/롤백 개념 없음)

**ledger item 구조**
```js
{
  actionType,              // "productionCompletion" / "refresh" / "processing" 등
  actionId,               // 해당 트랜잭션 문서 ID
  timestamp,              // 실행 시각
  items: [                // 변경된 재고 item 목록
    {
      collection,
      docId,
      field,              // "remaining" 또는 "currentQty"
      delta,
      before,
      after,
      stockUpdatedAtSnapshot
    }
  ]
}
```

**productionCompletion 문서 구조**
```js
{
  runDate: "2026-04-27",
  targetProductionDate: "2026-04-28",
  status: "completed",
  idempotencyKey: "productionCompletion:2026-04-27",
  ledgerId: "ledger_doc_id"  // stockLedger 문서 1개 참조
}
```
- 원육/봉투 차감은 runDate(오늘) 기준으로 처리
- 차감 대상 생산 조회는 targetProductionDate 기준

**롤백 처리**
- ledger items delta 반전 적용
- 내일생산불러오기 당시 차감분만 복원 (이후 수동조정분 유지)
- 새로고침 시: 기존 ledger 롤백 후 재차감하며 새 ledger 생성 (이전 ledger 무효화)

**재고 정합성 원칙**
- master 문서의 remaining/currentQty가 항상 권위있는 값
- meatLogs/bagLogs 등 이력과 불일치 시 master 문서 기준으로 판단
- 전처리 등록 시 냉동창고 자동 출고 ledger의 actionId는 해당 전처리 문서 ID
- 입고 예정 완료 시 원육(meatStocks.remaining)/봉투(bagTypes.currentQty)/계란(eggStock.currentQty) 증가분 모두 ledger 기록

**충돌 감지**
- 롤백 시 현재값이 ledger after와 다르면 충돌 판단
- 충돌 시 팝업:
  "내일생산불러오기 이후 [원료명] 재고가 변경된 이력이 있습니다.
  변경 전 수량으로 되돌리면 이후 수동 조정분은 유지되고,
  내일생산불러오기 당시 차감분만 복원됩니다.
  강제 복원하시겠습니까?"
  **예** / **아니오**

## 동결제품-분리작업 독립 원칙
- 동결제품 입고: 봉투에 포장 완료된 완제품 입고. 봉투 차감됨.
- 동결 분리작업: 포장 전 원물 상태 관리 (빵판/동결판)
- 두 메뉴는 완전 독립. 자동 연동 없음.

## 전처리/재포장 삭제 정책
- 당일 등록만 삭제 가능
- 삭제 시 ledger 기반 냉동창고/전처리 재고 롤백
- 이전 날짜 등록은 삭제 불가, 실수는 수동 조정으로 처리
- **권한**: `admin/office/production` 모두 가능 (v20 명시)

## 동결제품 입고 수정/삭제 정책

# 20. 재고관리 앱 연동 (v20 신설, v22 결정 8 추가, 미구현)

생산관리 앱과 별개의 재고관리 앱을 추후 개발 예정. 본 절은 운영자가 확정한 연동 정책을 기록한 것이며, 실제 구현은 **생산관리 앱 운영 시작 후 별도 묶음**으로 진행한다 (결정 4 = A).

## 결정 사항

### 결정 1 — 제품 마스터는 생산관리 앱 소유
- 재고관리 앱은 생산관리 앱의 `productId`를 참조만
- 새 제품 등록은 생산관리 앱에서 (레시피 등록 시 자연스럽게)
- 재고관리 앱은 읽기 전용

### 결정 2 — 카테고리도 생산관리 앱이 마스터
- 동결건조 종류 추가 (예: 트릿동결) 시 생산관리 앱에서 등록
- 재고관리 앱은 참조만

### 결정 3 — 판매 제품만 연동 대상
- 연동 범위: 생식 / 생식동결 / 텐더동결 / 추후 추가될 판매 제품
- 비연동: 재고관리 앱 단독의 supplies (브로슈어, 케이블 타이 등)
- supplies는 재고관리 앱이 단독으로 productId 발급

### 결정 4 — 진행 시점: A (운영 시작 후) — v22 재확인
- 생산관리 앱 운영 시작 후 별도 묶음으로 진행
- 재고관리 앱 개발은 그 이후
- 운영 1~2주 돌리며 발견되는 필드 변경 자연스럽게 반영 후 안정화된 마스터 위에서 개발
- **v22 보강**: 묶음 e 중 "운영 시작 전 진행" 권장 외부 문서가 작성되었으나 본 결정과 모순되어 폐기됨. 결정 4 유지 확정.

### 결정 5 — 재고관리 앱이 읽어야 하는 데이터
- Product 마스터 (`productId`, `productName`, `productCategory`, `species`, `productForm`, `unit` 등)
- ProductCategory 마스터 (`categoryCode`, `categoryName`, `inventoryRuleGroup`, `hasSampleSku`, `expirationRuleDays`)
- 생산 완료 데이터 (`productions` / `frozenLogs`) → IncomingStock으로 받음

### 결정 6 — 입고 흐름
1. 생산관리 앱에서 생산 완료 처리 → `status: sentToInventory`
2. 재고관리 앱이 폴링 또는 webhook으로 받음 → IncomingStock 생성 (`sourceApp: productionApp`, `status: pending`)
3. 재고관리 앱 admin이 확인 → StockLot에 반영 → `status: confirmed`
4. 생산관리 앱에 역방향 신호: `inventoryReceived` 상태 업데이트

### 결정 7 — 본품 해체 흐름: 옵션 1로 시작
- **초기**: 외부 전달 (전화, 메신저) — 시스템 연동 없음
- **추후 옵션 2 발전**: 해체 빈도 보고 결정. 매일 발생이면 옵션 2 도입 (생산관리 앱에 "해체 작업 의뢰" 메뉴 추가)

### 결정 8 — 메인 카드 입력 모달 통합 (v22 신설, C안)

운영 시작 후 재고관리 앱 묶음과 같이 진행:

#### 변경 내용
- 현재: 메인 화면 생산 카드 = 보기 전용
- 변경 후: 카드 클릭 → **생산 입력 모달** (생식/생식동결/텐더동결 3분기)

#### 분기 3개
| 카테고리 | 클릭 시 모달 | 입력 필드 |
|---|---|---|
| 생식 (raw) | "실제 생산량 입력" | 계육박스 N + 잔여 낱팩 M + 제조일 |
| 생식동결 (freezeDry, `requiresSeparation:true`) | "빵판 입고" | 빵판 N개 + 날짜 + 담당자 |
| 텐더동결 (freezeDry, `requiresSeparation:false`) | "동결판 입고" | 동결판 N판 + 날짜 + 담당자 |

#### 생식 환산 로직
```
총 팩수 = (계육박스 수 × 환산비) + 잔여 낱팩
  - 고양이 환산비: 135팩/계육박스 (packsPerChickenBox)
  - 강아지 환산비: 90팩/계육박스

총 팩수에서:
  - 20팩 단위 = 1박스 (packsPerBox=20)
  - 잔여 10팩 = 0.5박스
  - 10팩 미만 = 샘플 팩
```

#### 메뉴 7번/8번 기존 버튼 처리
운영자 결정 필요 (재고관리 앱 묶음 시작 시):
- (가) 메뉴 버튼 + 메인 카드 둘 다 유지
- (나) 메뉴 버튼 제거, 메인 카드만 — 카드 없는 임시 입고는 메인에서 카드 추가 후 진입
- (다) 메뉴 버튼 유지 + 메인 카드는 해당 메뉴로 이동

### 기타 결정 (재고관리 앱 묶음 시작 시 정식 spec 반영 예정)

| 항목 | 결정 |

| 연동 방식 | A — Firestore 직접 공유 |
